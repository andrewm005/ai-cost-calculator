About AI Cost Calculator

A free, no-signup AI token-cost calculator. Last updated 2026-06-23

I built this because my OpenRouter bill kept surprising me and the invoice never explained why. AI Cost Calculator reads OpenRouter's live pricing feed, lets you pick a model and a workload, and gives you a dollar figure you can quote without doing the arithmetic by hand.

About me

Andrew Morgado

Data Science student, learning AI

I'm a data science student working on my bachelor's degree. I have been writing code since I was a kid, starting with Scratch, then BASIC, then Python, and the through-line from then to now is the same one: I like building tools that answer a question I keep asking. AI is the most important shift I will see in my lifetime, and tokenomics is the part of it most people skip over. The price of a token decides who can ship, what they ship, and whether what they ship survives contact with real users. I built AI Cost Calculator because I needed that number myself, and I plan to keep updating it as the economics of AI stop being a niche question and start being baseline literacy.

I run agentic coding sessions on OpenClaw and Hermes through OpenRouter, where every code-generation loop pulls a fresh batch of tokens from a model I picked from a list of 300+ names I barely understood, and every loop has two costs: tokens going in and tokens coming out. Some loops cost a few cents. Some cost dollars. The total adds up faster than I expected.

OpenRouter's invoice tells me what I spent last month. It does not tell me which model was responsible, or whether switching from Claude Sonnet to GPT-4o-mini would have saved me anything. So I opened tabs for the pricing pages, copied them into another AI, and asked it to compare. Then I did it again for Gemini. Then for DeepSeek. Each round of asking cost another batch of tokens, and by the time I had a rough answer I'd spent more time and more money figuring out the savings than I would have saved by switching in the first place.

AI Cost Calculator is what I built to skip that loop. Pick a model. Pick a workload. Get a number in dollars. Compare two models side by side. The math is the same math another AI would have run, just faster, and the price comes from OpenRouter's live feed instead of stale blog screenshots.

LinkedIn

The site started as a one-evening hack to stop me from redoing the same arithmetic every time I wanted to swap a model. It now covers 80 curated models, the ones real developers ship against, plus the long tail: open-weights, niche labs, auto-routers. The kind of stuff the official per-vendor pricing pages don't list in one place.

Pricing is live from OpenRouter. OpenRouter's feed refreshes every six hours, which means the numbers here are never more than a few hours behind vendor list prices. Hand-curated entries for the major hosted APIs get cross-checked against each vendor's pricing page on the same cycle, which matters because some providers update their public pricing faster than OpenRouter does.

The math is open. Every multiplier, every preset, the calculator source. All linked from the main page. I do not track you, sign you up, run analytics, or set cookies. The privacy policy lists everything the site intentionally does not do. The site is boring on purpose. It is a tool, not a funnel.

The FAQ lives in the calculator page. The raw model list is public JSON, which matters if you want to build on top of this instead of clicking through it.

Methodology

Data sources. I pull prices from OpenRouter's /api/v1/models endpoint. OpenRouter aggregates most hosted providers (OpenAI, Anthropic, Google, Mistral, DeepSeek, Meta, xAI, Cohere, Qwen, Perplexity, and the long tail) and exposes them through one consistent schema. That's why I anchor on it. Hand-curated entries in config/pricing.json cover the direct-API surface for models OpenRouter doesn't carry.

Refresh cadence. A background loop re-fetches the OpenRouter feed every six hours (configurable via OPENROUTER_REFRESH_SECONDS) and rewrites the cache in place. Every generated model page carries a Last updated timestamp tied to the most recent generation. The main calculator hits the live cache on every request, so the numbers you see are at most a few hours behind the vendor list price.

What the calculator covers. Input and output token cost. Cached input, where the provider supports it (read from cached_input_per_1m). Reasoning tokens (reasoning_per_1m for o-series, R1, Claude thinking, Gemini thinking models). Flat per-call tool cost (tool_call_cost). Image input at a flat per-image estimate. Agentic overhead: a 2,000-token system prompt, 5 tool calls per loop, and a 1.4× retry multiplier that matches what agent frameworks actually observe in production.

What it does not cover. Per-image variance across providers. I use a flat per-image estimate, not the actual multi-resolution cost. Batch API discounts (typically 50 % off for OpenAI and Anthropic). Fine-tuning costs. Self-hosted GPU cost (covered separately at the /calculate/local endpoint, which factors GPU rental plus power into a per-token rate). Enterprise contracts. Regional pricing variations. Any post-hoc credits or rebates. The page you got the number from was generated against the headline on-demand rate. Run your own batch test before quoting a customer.

How errors are handled. If OpenRouter is unreachable on the scheduled refresh, the calculator keeps serving the last cached price and surfaces a last verified X hours ago banner. I never invent prices. If a model disappears from the upstream feed, it disappears from the catalog on the next refresh. I don't retroactively patch in a replacement.

What I don't do. No tracking. No signup. No analytics. No cookies. See the privacy policy for the full list of what's intentionally not on this site.

Scope (why 80 models and not 334). The /models/ directory is a curated 80, not a dump of every OpenRouter listing. I chose depth over breadth. Every page here is for a model real developers search for. Every page has a real FAQ, a real workload-cost breakdown, a real related-models block. The full OpenRouter feed powers the live /models API endpoint, which the calculator itself uses, but does not produce 254 thin pages that add no value beyond the main calculator.
