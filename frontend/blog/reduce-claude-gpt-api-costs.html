How to Reduce Claude and GPT API Costs by 60%

Most Claude and GPT API bills are 40-60% higher than they need to be. The reason is not vendor pricing. It is that the default call pattern ignores five multipliers that everyone publishes and most teams never turn on: prompt caching, model tiering, prompt trimming, batch processing, and the system prompt. This article walks through each one with real numbers from the OpenRouter feed and shows what the bill looks like before and after.

Tactic 1: Turn on prompt caching

Prompt caching is the largest single line item most teams miss. Anthropic's cache rate is 10% of the input rate. Claude Sonnet 4 lists input at $3.00 per 1M tokens; the cached input is $0.30 per 1M. If your application ships a 2,000-token system prompt on every call, the system prompt alone is $0.006 per call at the input rate and $0.0006 per call at the cache rate, a 10x reduction. At one million calls per month, the system prompt is $6,000 at the input rate and $600 at the cache rate.

OpenAI publishes cache rates at 50% off input on supported models. GPT-4o mini is $0.15 input and $0.075 cache per 1M tokens. GPT-4.1 is $2.00 input and $0.50 cache. Google's Gemini 2.5 family has implicit caching that activates on prompts above a token threshold, with no code change. The catch for every provider: the cache is prefix-based, so the cacheable block must be at the start of the prompt. The prompt caching article has the structural details.

Tactic 2: Tier your models

Not every call needs the flagship. A production pipeline usually has a small number of complex calls (deep reasoning, code review, long synthesis) and a large number of simple calls (classification, extraction, short replies). Routing them all through the flagship is the most expensive way to do it.

Concrete numbers from the OpenRouter catalog: Claude Opus 4 is $15.00 input and $75.00 output per 1M tokens. Claude Sonnet 4 is $3.00 and $15.00, a 5x reduction on input and output. Claude Haiku 4.5 is $1.00 and $5.00, a 15x reduction. For workloads where Haiku is good enough (which is most classification, simple extraction, and short chat replies), the bill drops by an order of magnitude.

Same pattern on OpenAI. GPT-4o is $2.50 and $10.00. GPT-4o mini is $0.15 and $0.60, a 17x reduction. See the GPT-4o vs GPT-4o mini comparison for a workload-level view, and the Opus 4 vs GPT-4o comparison for a flagship-to-flagship view. The right pattern is a router that classifies the request and picks the model.

Tactic 3: Trim the prompt

Prompts grow. A persona section that started at 200 tokens gains "edge case handling" clauses, then a "tone guidance" section, then a list of disallowed behaviors. Six months in, the system prompt is 2,500 tokens and shipping on every call. Trim aggressively.

Two specific moves. First, move examples out of the system prompt and into a one-shot user message. Anthropic's prompt cache still hits the prefix if the examples are stable, but if examples vary by request, you save the input cost by moving them out. Second, remove redundant instructions. "Be helpful, accurate, and concise" adds nothing the model doesn't already do; cut it.

A reasonable goal: keep the system prompt under 800 tokens for short-form workloads, under 1,500 for long-context workloads. If yours is over 2,000 tokens, there is probably a 30% trim available without losing capability. The system prompt tokens article walks through the per-call math.

Tactic 4: Use the Batch API for async work

OpenAI's Batch API charges 50% of the synchronous rate for jobs with up to 24-hour turnaround. For workloads that don't need a real-time response. Bulk classification, overnight document processing, eval generation, embedding backfills. Batch is the cheapest reliable path on the OpenAI stack.

Anthropic's Message Batches API offers a similar 50% discount on Claude calls for batch processing. Google publishes a Batch mode for Gemini with comparable savings. Most production systems have a meaningful fraction of their traffic that can move to batch without changing the user experience. If 30% of your OpenAI bill is bulk classification that runs nightly, that 30% drops by 50% immediately.

Tactic 5: Cap the output

Output tokens are 3-5x more expensive than input tokens on most models. Claude Sonnet 4 is $3.00 input and $15.00 output. GPT-4o is $2.50 input and $10.00 output. Gemini 2.5 Pro is $1.25 input and $10.00 output. A model that runs to its maximum output length because no stop condition was set can produce 4,000 tokens when 400 would have answered the question.

The fix is mechanical: set a `max_tokens` cap on every call that fits the expected output shape. For classification, 50 tokens. For short replies, 250 tokens. For long-form generation, the cap matches the deliverable. The model will stop naturally when it's done; the cap is a ceiling for runaway cases. Most teams find their output token averages drop by 30-50% after they add caps.

What about quality?

The legitimate concern with each of these tactics is quality regression. Caching the system prompt should not change the output because the cached prefix is the same prefix the model was already seeing. Tiering models down means accepting that the smaller model will be weaker on the hardest fraction of calls. a router that flags uncertain cases and reroutes them to the flagship fixes most of this. Trimming the prompt is the most dangerous move and should be evaluated before shipping. Batch API has no quality impact; the model and inputs are identical. Capping output has no quality impact for normal calls; it only protects against runaway cases.

Frequently asked questions

What's the single biggest cost saver for Claude and GPT API bills?

Prompt caching. Anthropic's cache rate is 10% of input (Claude Sonnet 4: $0.30 cache vs $3.00 input per 1M). OpenAI's cache rate is 50% off input on supported models. For any application with a long system prompt that ships on every call, prompt caching is the difference between a small bill and a six-figure annual line item.

Is Haiku really 15x cheaper than Opus for similar tasks?

On the rate card, yes. Claude Opus 4 is $15.00 per 1M input and $75.00 per 1M output. Claude Haiku 4.5 is $1.00 input and $5.00 output. That is 15x cheaper on input and output. In practice Haiku is not a drop-in for Opus on tasks that require deep reasoning, but for classification, extraction, summarization, and short-form generation it's a meaningful cost cut at acceptable quality.

Does the OpenAI Batch API cut costs by 50%?

Yes. OpenAI's Batch API accepts asynchronous jobs with a 24-hour turnaround and charges 50% of the synchronous rate. For workloads that don't need real-time responses (bulk classification, overnight document processing, eval generation), Batch is the cheapest reliable path on the OpenAI stack.

Should I downgrade from GPT-4o to GPT-4o mini?

It depends on the workload. GPT-4o mini at $0.15/$0.60 is 17x cheaper on input and output than GPT-4o at $2.50/$10.00. For chat replies, simple extraction, and most short-form generation, mini is good enough and the savings are immediate. For long-context synthesis, complex code review, or any task where the flagship's stronger reasoning is the point, mini is not a drop-in.
