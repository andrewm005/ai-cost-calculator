How Much Does an Agentic Workflow Actually Cost?

A chat-shaped call has one input and one output. An agentic workflow has N inputs and N outputs, where N is the number of model calls the agent makes to complete one user request. A coding agent that loops five times to fix a bug is five model calls. The bill is roughly five times a single chat call, with two multipliers on top: the conversation history grows with every loop, and the model might emit reasoning tokens on every loop. This article walks through the math on a real agent.

What "agentic" actually means

An agentic workflow is any LLM application where the model decides which tool to call next, observes the result, and calls another tool. The decision loop is the agent. The difference from a single chat call is that the agent can take multiple steps, the conversation grows with each step, and the bill scales with the step count.

The popular agent frameworks all follow the same pattern. The model gets a system prompt with a list of available tools, looks at the user request, picks a tool, reads the result, picks the next tool, and loops until it decides the task is done. Most production agents loop 5-15 times per user request. The aggressive ones (a research agent doing web search + synthesis) can loop 30+ times.

The representative agent

The example is a coding agent modeled on Claude Code or Cursor's agent mode. The user asks the agent to fix a bug in a small project. The agent's job: read the file, find the bug, propose a fix, run the test suite, verify the fix, and write a commit message. Six tool calls minimum, often eight to twelve.

The agent's system prompt is roughly 1,500 tokens (persona, tool definitions, project conventions). Each tool call adds a tool call request and a tool result to the conversation history. The model is Claude Sonnet 4 at $3.00 input and $15.00 output per 1M tokens. The cache is enabled on the system prompt.

The loop-by-loop math

Loop 1: the model receives the 1,500-token system prompt (cache hit at $0.30 per 1M = $0.00045), a 200-token user message ($0.0006), and produces a 300-token response including a tool call request ($0.0045). Loop 1 cost: $0.00555. After the tool call, the conversation history is 1,700 tokens for system + 200 user + 300 assistant + 50 tool result = 2,250 tokens.

Loop 2: the model receives 2,250 tokens of context ($0.000675) and produces another 300-token response with a tool call. Loop 2 cost: $0.00518. After the tool call, conversation history is 2,650 tokens.

Loop 3: 2,650 × $3.00/M + 300 × $15.00/M = $0.00795 + $0.0045 = $0.01245. Wait, that's higher than loops 1 and 2. The reason: the conversation history is no longer mostly cache. The first 1,500 tokens still hit the cache, but the next 1,150 tokens (user message + prior assistant + tool calls + tool results) are variable and bill at the full input rate. The cost grows linearly with the conversation history.

By loop 6, the conversation history is 5,000 tokens. The loop's input cost is $0.015 ($0.00045 cache + $0.01455 variable input at the full rate). By loop 10, history is 7,500 tokens and the loop's input cost is $0.0225. Output stays constant at $0.0045 per loop.

The full task

A 10-loop agent on Claude Sonnet 4 with a 1,500-token cached system prompt: roughly $0.12 input + $0.045 output = $0.165 per task. At 10,000 tasks per month, that's $1,650. A user-shaped chat call on the same model is $0.0105; 10,000 chat calls is $105. The agentic workflow is 15x more expensive per user request.

That ratio scales with the loop count. A 5-loop agent is roughly $0.08 per task. A 20-loop agent is $0.40 per task. The right planning assumption for an agentic product is that each user request costs 10-30x a single chat call.

Reasoning tokens on top

If the agent runs on a reasoning model such as OpenAI o1, Claude with extended thinking, Gemini 2.5 Pro, DeepSeek R1. Every loop pays the reasoning tax. A 10-loop agent on OpenAI o1 with 2,000 reasoning tokens per loop is paying $0.12 per loop just for reasoning tokens. Total per task: $1.20 before any visible output.

DeepSeek R1 at $0.70/$2.50 per 1M tokens is the cost-conscious pick for reasoning agents. The same 10-loop agent with 2,000 reasoning tokens per loop on R1 is $0.05 per task in reasoning tokens. The reasoning token article has the full breakdown.

Conversation history is the hidden tax

The single biggest cost driver in an agentic workflow is the conversation history re-billed on every loop. Most agent frameworks (LangChain, AutoGen, CrewAI) send the full transcript on every model call. By loop 10, 70% of the input tokens are prior transcript, not new information.

Three mitigations. First, cap the conversation history at the last N tokens (typical: 8,000 to 16,000). Older context is summarized into a compact state and re-injected. Second, use a sliding window of the most recent tool results and drop the rest. Third, use Anthropic's prompt caching aggressively on the system prompt and tool definitions, both are stable across loops. The Sonnet 4 vs Haiku 4 comparison is useful when the easy loops in an agent can be routed to the cheaper model.

Cheaper model on the easy loops

A typical agent has 2-3 hard loops (the ones that need reasoning about the codebase, planning the fix, writing the patch) and 5-8 easy loops (file lookups, grep, running tests, parsing output). The hard loops need the flagship. The easy loops can run on a cheaper model that is faster and costs less per call.

A two-model agent on Claude Sonnet 4 (hard loops) and Claude Haiku 4.5 (easy loops): roughly 50% reduction on the easy-loop portion of the bill. On a 10-loop agent that does 3 hard loops and 7 easy loops, the savings are roughly $0.05 per task. Meaningful at 10,000 tasks/month.

The wrong way to estimate

A common mistake is to estimate the cost of an agentic product by multiplying a single chat call by the per-user request count. That gives you the wrong answer by 10-30x. The right estimate is per-task cost (which includes all the loops) multiplied by the task count. The per-task cost depends on the loop count, the model, the cache hit rate, and the conversation growth pattern. None of those are visible in a single-call estimate.

Frequently asked questions

How much does a single agent loop cost?

A single agent loop is one model call plus one tool call. On Claude Sonnet 4 at $3.00 input / $15.00 output, a typical loop with 4,000 input tokens and 1,000 output tokens is $0.012 + $0.015 = $0.027. An agent that loops ten times to complete a task is $0.27, before any reasoning tokens.

Why does conversation history matter for cost?

Most agent frameworks send the full conversation history on every model call. That includes the system prompt, every prior user message, every prior assistant response, every tool call request, and every tool result. By turn five of an agent loop, the conversation is often 10x the size of the original prompt.

Do reasoning models make agents more expensive?

Yes, often by 3x to 10x. A reasoning model like OpenAI o1 at $60 per 1M output spends 1,000 to 3,000 reasoning tokens per loop before producing the visible answer. An agent that loops ten times on o1 is paying $6 to $18 just for reasoning tokens.

What's the cheapest way to run an agent?

Use a cheap, fast model for the easy loops (file lookups, simple tool calls, classification) and the flagship only for the loops that genuinely need it. Use prompt caching aggressively on the system prompt and tool definitions. Cap the conversation history so the model does not re-bill the entire prior transcript on every turn.
