What Is a Reasoning Token? o1, o3, and DeepSeek-R1 Explained

Reasoning tokens are the chain-of-thought tokens a reasoning model emits between your prompt and the final answer you see. They are real tokens that the model generates, they cost real money, and they are not shown in the visible response. If you are paying an LLM API bill and you use any model whose name starts with o (OpenAI), R1 (DeepSeek), or has 'thinking' in its description, you are paying for reasoning tokens.

The short version

A reasoning token is one unit of internal deliberation that the model writes to itself before producing the user-visible answer. On OpenAI's o1, a typical math problem will trigger 500 to 2,000 reasoning tokens before the model commits to a final answer. The visible answer might be 200 tokens; the bill is for 700 to 2,200 tokens. OpenAI prices o1 at $15.00 per 1M input tokens and $60.00 per 1M output tokens on OpenRouter, and reasoning tokens fall in the output bucket.

The same pattern holds across providers, with different names for the same idea. Anthropic calls it "extended thinking." Google calls it "thinking" and surfaces it through the reasoning_per_1m field on Gemini 2.5 Pro. DeepSeek calls its R1 model exactly what it is: a model that thinks out loud, with the thinking billed as output. The design pattern is the same. Produce a chain of intermediate tokens before the final answer, but the pricing differs enough that you should know the rates before picking one.

Why reasoning exists

A standard language model produces one token at a time, each conditioned on the prompt and every token it has already produced. That works well for fluent text and recall-style questions, but it fails on tasks that require backtracking, search, or verification. "What is 37 times 43?" is easy if the model can think "37 times 40 is 1480, plus 37 times 3 is 111, total 1591" before writing the final answer. Without the thinking step, the model has to commit to a single token sequence with no intermediate scratchpad, and it makes mistakes that a calculator would not.

The reasoning models are trained to write out that scratchpad as part of their output. The chain-of-thought acts as a working memory that the model can read back to itself when it commits to the next token. On math benchmarks, this is the difference between random guessing on competition problems and solving a meaningful fraction of them. The model's "intelligence" on hard tasks is largely a function of how many reasoning tokens it's willing to spend before it has to commit to an answer.

The bill

Reasoning tokens are billed as output, full stop, on every major provider as of mid-2026. The user-visible answer might be 300 tokens; the bill might be 3,300 tokens if the model spent 3,000 reasoning tokens working up to it. For workloads where the visible answer is the deliverable (a one-paragraph email, a single SQL query). Reasoning tokens can multiply the effective cost-per-output by 5x to 10x.

A worked example with OpenAI o1: a developer asks o1 to refactor a 200-line function. The input prompt is the function (about 800 tokens) plus the request (50 tokens). The model thinks through the refactor for 1,500 reasoning tokens. The final answer is the refactored function at 250 tokens. Total billed tokens: 850 input + 1,750 output (1,500 reasoning + 250 visible). Cost: $0.0128 input + $0.105 output = $0.1178 per call. The same task on GPT-4o at $2.50/$10.00 is $0.0021 input + $0.0025 output = $0.0046 per call. The reasoning model is 25x more expensive on this specific task, and most of that is reasoning tokens.

The same trade-off with DeepSeek R1: $0.70 input, $2.50 output. A 1,500-reasoning-token, 250-visible-token answer on a 850-token prompt runs $0.0006 input + $0.0044 output = $0.0050 per call. Twenty-four times cheaper than o1 on the same shape, and the reasoning quality is comparable on most non-frontier tasks. That's why DeepSeek R1 is the model the cost-conscious shops reach for when they need reasoning but not o1-class frontier performance.

When reasoning helps

Reasoning tokens pay for themselves on tasks where the model would otherwise commit to a wrong answer. The clearest cases are math, multi-step logic, code with non-obvious bugs, and any problem that requires the model to consider and rule out alternatives before committing. On a calculus problem, a chemistry synthesis question, or a SQL query that needs three joins, the reasoning tokens are doing the work; without them, the model would output a confident wrong answer. See the o3 vs Claude Sonnet 4 comparison for a side-by-side view of a mid-tier reasoning model against a flagship non-reasoning one.

A subtler case is tasks where the visible answer needs to be precise in a way that the model can't reliably produce in a single pass. Generating a legal contract clause, producing a unit-tested code function, or writing a multi-step data pipeline. These all benefit from a chain-of-thought that lets the model verify its own work as it goes. The reasoning tokens are an insurance policy against the embarrassing failure mode of an LLM producing fluent, plausible, but wrong output.

When reasoning does not help

Reasoning tokens are wasted on tasks that are simple enough that the model can answer correctly in one pass. Summarization, classification, extraction, sentiment analysis, and most chat replies do not benefit from thinking out loud. If you ask a reasoning model to summarize a document, it will sometimes spend hundreds of reasoning tokens second-guessing its own summary, and the visible answer is no better than what a non-reasoning model would have produced in a single pass.

Routing matters here. A production system should not route every request through a reasoning model. The right pattern is a router that classifies the request. Short chat and extraction go to a cheap, fast model; multi-step reasoning goes to a reasoning model. OpenAI's o3 at $2.00 input / $8.00 output is the mid-tier reasoning option that makes sense for problems where reasoning helps but you don't need o1. o3-mini at $1.10 / $4.40 is the budget option for "I want some thinking, but I don't want to pay for it." The non-reasoning models (GPT-4o, Claude Sonnet 4, Gemini 2.5 Flash) are the right answer for everything else.

How to control the thinking budget

Most reasoning models let you control how aggressively they think. OpenAI exposes a `reasoning_effort` parameter on o1 and o3 with values `low`, `medium`, and `high`. Setting it to `low` produces shorter reasoning chains and a cheaper bill, at some cost to accuracy on hard problems. Anthropic exposes a `thinking_budget` parameter on extended thinking Claude. You can cap the reasoning at, say, 2,000 tokens, and the model will commit to its final answer when it hits that cap. Google's Gemini 2.5 Pro has a thinking budget in the API call.

The right default for a production system is to set a budget that matches the task. For a coding assistant that needs to think through a multi-step refactor, 4,000 reasoning tokens is a reasonable default. For a customer support chat that needs just enough thinking to verify a policy lookup, 500 tokens is plenty. The model will still spend what it spends. The budget is a ceiling, not a target. but it stops runaway chains from generating 10,000 reasoning tokens on a single user request.

Summary

Reasoning tokens are a useful tool for hard tasks and a costly one for easy ones. The price of admission is real: o1 at $60 per 1M output tokens means a 2,000-reasoning-token call adds $0.12 to the bill before the model writes the visible answer. DeepSeek R1 at $2.50 per 1M output tokens gives you the same chain-of-thought pattern at roughly 1/24th the cost, with a quality gap on the frontier problems. Pick the reasoning model for the right tasks, set a thinking budget, and use a non-reasoning model for everything else.

Frequently asked questions

What is a reasoning token?

A reasoning token is one of the internal chain-of-thought tokens that a reasoning model like OpenAI's o1 or o3, Anthropic's Claude with extended thinking, Google's Gemini 2.5 Pro, or DeepSeek's R1 emits before producing the visible answer. Reasoning tokens are billed as output but are not shown to the user.

How much does reasoning cost on o1?

OpenAI's o1 is priced at $15.00 per 1M input tokens and $60.00 per 1M output tokens on OpenRouter. Reasoning tokens count as output, so a long chain-of-thought can push a single o1 call to several cents even when the visible answer is short.

Is DeepSeek R1 cheaper than o1?

Yes, substantially. DeepSeek R1 is priced at $0.70 per 1M input tokens and $2.50 per 1M output tokens. That's roughly 20x cheaper than o1 on input and 24x cheaper on output. The catch is reasoning quality on hard math and coding tasks, where o1 still has a meaningful lead on the most difficult problems.

Can I turn reasoning off?

On OpenRouter you can switch from a reasoning model to a non-reasoning one. o1 to GPT-4o, for example. Within a reasoning model, Anthropic's Claude extended thinking lets you set a thinking budget in tokens. OpenAI's o-series gives you a 'reasoning_effort' parameter (low, medium, high) that controls how aggressively the model thinks.

Why don't all providers publish a separate reasoning price?

Some don't separate it because reasoning tokens fall into the same output bucket as the final answer. OpenAI's o-series bills reasoning at the output rate. Anthropic's extended thinking uses a per-token thinking rate that mirrors the output rate. Google publishes a reasoning_per_1m field but it often arrives null in cached views.
