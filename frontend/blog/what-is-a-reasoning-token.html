What Is a Reasoning Token? o1, o3, and DeepSeek-R1 Explained

Reasoning tokens are billed as output but are not shown to the user. A 2,000-reasoning-token chain on o1 can multiply the effective per-call cost by 5x to 10x. The question is not whether to use reasoning models. It is how to cap the chain. Reasoning models emit an internal scratchpad before the visible answer — OpenAI's o-series, Anthropic's Claude with extended thinking, Google's Gemini 2.5 Pro, DeepSeek's R1 — and every token of that scratchpad lands on the bill under output. Any team paying for o1 calls without a thinking budget is paying for reasoning it cannot see.

When reasoning pays for itself

Reasoning tokens earn their keep on tasks where the model would otherwise commit to a wrong answer. The clearest cases are math, multi-step logic, code with non-obvious bugs, and any problem that requires the model to consider and rule out alternatives before committing. On a calculus problem, a chemistry synthesis question, or a SQL query that needs three joins, the reasoning tokens are doing the work. Without them, the model outputs a confident wrong answer. A study found that frontier reasoning models solve 70%+ of competition math problems with extended thinking enabled and under 30% without. This suggests the chain-of-thought is doing real computational work, not just hedging.

A subtler case is tasks where the visible answer needs to be precise in a way the model cannot reliably produce in a single pass. Generating a legal contract clause, producing a unit-tested code function, writing a multi-step data pipeline — these all benefit from a chain-of-thought that lets the model verify its own work. The reasoning tokens are an insurance policy against the embarrassing failure mode of an LLM producing fluent, plausible, but wrong output. Clearly, reasoning is the right tool on tasks where a wrong answer has a real cost.

When reasoning is wasted spend

Reasoning tokens are wasted on tasks simple enough that the model can answer correctly in one pass. Summarization, classification, extraction, sentiment analysis, and most chat replies do not benefit from thinking out loud. If a reasoning model is asked to summarize a document, it sometimes spends hundreds of reasoning tokens second-guessing its own summary, and the visible answer is no better than what a non-reasoning model would have produced in a single pass.

Routing matters here. A live system should not route every request through a reasoning model. The right pattern is a router that classifies the request: short chat and extraction go to a cheap, fast model; multi-step reasoning goes to a reasoning model. OpenAI's o3 at $2.00 input / $8.00 output is the mid-tier reasoning option that makes sense for problems where reasoning helps but o1 is overkill. o3-mini at $1.10 / $4.40 is the budget tier for "I want some thinking, but I do not want to pay for it." The non-reasoning models (GPT-4o, Claude Sonnet 4, Gemini 2.5 Flash) are the right answer for everything else.

The bill on a real refactor

A worked example with OpenAI o1: a developer asks o1 to refactor a 200-line function. The input prompt is the function (about 800 tokens) plus the request (50 tokens). The model thinks through the refactor for 1,500 reasoning tokens. The final answer is the refactored function at 250 tokens. Total billed tokens: 850 input + 1,750 output (1,500 reasoning + 250 visible). Cost: $0.0128 input + $0.105 output = $0.1178 per call. The same task on GPT-4o at $2.50/$10.00 is $0.0021 input + $0.0025 output = $0.0046 per call. The reasoning model is 25x more expensive on this specific task, and most of that is reasoning tokens.

The same trade-off with DeepSeek R1: $0.70 input, $2.50 output. A 1,500-reasoning-token, 250-visible-token answer on an 850-token prompt runs $0.0006 input + $0.0044 output = $0.0050 per call. Twenty-four times cheaper than o1 on the same shape, with reasoning quality comparable on most non-frontier tasks. That is why DeepSeek R1 is the model the cost-conscious shops reach for when they need reasoning but not o1-class frontier performance. The o3 vs Claude Sonnet 4 comparison puts a mid-tier reasoning model against a flagship non-reasoning one.

How the pricing actually works by provider

Reasoning tokens are billed as output, full stop, on every major provider in mid-2026. OpenAI's o1 lists $15.00 input and $60.00 output per 1M tokens, and reasoning falls in the output bucket. Anthropic's extended thinking bills at the output rate, which on Claude Sonnet 4 is $15.00 per 1M. Google's Gemini 2.5 Pro publishes a reasoning_per_1m field that mirrors the $10.00 output rate. DeepSeek R1 publishes $0.70 input and $2.50 output, with reasoning in the output bucket. The user-visible answer might be 300 tokens; the bill might be 3,300 tokens if the model spent 3,000 reasoning tokens working up to it. When the visible answer is the answer is the deliverable (a one-paragraph email, a single SQL query), reasoning tokens can multiply the effective cost-per-output by 5x to 10x.

How to control the thinking budget

Most reasoning models expose a parameter that caps the chain. OpenAI publishes a `reasoning_effort` parameter on o1 and o3 with values `low`, `medium`, and `high`. Setting it to `low` produces shorter reasoning chains and a cheaper bill, at some cost to accuracy on hard problems. Anthropic exposes a `thinking_budget` parameter on extended thinking Claude. You can cap the reasoning at, say, 2,000 tokens, and the model commits to its final answer when it hits that cap. Google's Gemini 2.5 Pro has a thinking budget in the API call. The right default for a live system is to set a budget that matches the task. For a coding assistant that needs to think through a multi-step refactor, 4,000 reasoning tokens is a reasonable default. For a customer support chat that needs just enough thinking to verify a policy lookup, 500 tokens is plenty.

The model will still spend what it spends. The budget is a ceiling, not a target. But the ceiling stops runaway chains from generating 10,000 reasoning tokens on a single user request — a real failure mode when the prompt is open-ended. Obviously, the budget is the difference between a predictable line item and an open-ended one.

Why reasoning exists

A standard language model produces one token at a time, each conditioned on the prompt and every token already produced. That works well for fluent text and recall-style questions, but it fails on tasks that require backtracking, search, or verification. "What is 37 times 43?" is easy if the model can think "37 times 40 is 1480, plus 37 times 3 is 111, total 1591" before writing the final answer. Without the thinking step, the model has to commit to a single token sequence with no intermediate scratchpad, and it makes mistakes a calculator would not. The reasoning models are trained to write out that scratchpad as part of their output. The chain-of-thought acts as working memory the model can read back to itself when it commits to the next token. On math benchmarks, this is the difference between random guessing on competition problems and solving a meaningful fraction.

Frequently asked questions

What is a reasoning token?

A reasoning token is one of the internal chain-of-thought tokens that a reasoning model like OpenAI's o1 or o3, Anthropic's Claude with extended thinking, Google's Gemini 2.5 Pro, or DeepSeek's R1 emits before producing the visible answer. Reasoning tokens are billed as output but are not shown to the user.

How much does reasoning cost on o1?

OpenAI's o1 is $15.00 per 1M input tokens and $60.00 per 1M output tokens in mid-2026. Reasoning tokens count as output, so a long chain-of-thought can push a single o1 call to several cents even when the visible answer is short.

Is DeepSeek R1 cheaper than o1?

Yes, substantially. DeepSeek R1 is $0.70 per 1M input tokens and $2.50 per 1M output tokens, roughly 20x cheaper than o1 on input and 24x cheaper on output. The trade-off is reasoning quality on the hardest math and coding tasks.

Can reasoning be turned off?

Yes. Switch from a reasoning model to a non-reasoning one for workloads that do not need it (o1 to GPT-4o, for example). Within a reasoning model, Anthropic's extended thinking exposes a thinking_budget; OpenAI's o-series exposes reasoning_effort (low, medium, high).

Why do not all providers publish a separate reasoning price?

Some do not separate it because reasoning tokens fall into the same output bucket as the final answer. The bill arrives under output either way.
