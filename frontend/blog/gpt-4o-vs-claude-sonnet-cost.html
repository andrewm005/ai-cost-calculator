GPT-4o vs Claude Sonnet 4: Full Cost Breakdown

GPT-4o and Claude Sonnet 4 are the two flagship models most teams compare for general-purpose workloads. The headline rates say GPT-4o is cheaper; the cache rates and context window say Claude Sonnet 4 is more capable on long-context workloads. This article runs both models through five realistic workloads and shows what the bill looks like in each case, with no multipliers or hand-waving.

The headline rates

On OpenRouter as of mid-2026, GPT-4o is $2.50 per 1M input tokens and $10.00 per 1M output tokens. Claude Sonnet 4 is $3.00 per 1M input and $15.00 per 1M output. The input gap is 20%; the output gap is 50%. On a chat-shaped call with equal input and output, GPT-4o runs roughly 29% cheaper.

But the headline rate is the input to a real estimate, not the estimate itself. Three multipliers shape the bill: prompt caching, context window, and the typical workload shape. Prompt caching alone can flip the comparison.

The cache rates

GPT-4o does not publish a cached_input_per_1m rate on OpenRouter as of mid-2026. OpenAI supports prompt caching on GPT-4o mini and the GPT-4.1 family, but not on the base GPT-4o. Claude Sonnet 4 publishes a cache rate of $0.30 per 1M, which is 10% of its $3.00 input rate. For any application that ships a long system prompt on every call, Claude Sonnet 4's cache rate is the relevant input rate, and GPT-4o's missing cache rate means there is no cache to compare against.

A worked example. A 2,000-token system prompt shipped on every call, plus a 300-token user message. On GPT-4o the full 2,300 tokens bill at $2.50 per 1M = $0.00575 per call. On Claude Sonnet 4 with the system prompt cacheable, the 2,000 system tokens bill at $0.30 per 1M ($0.0006) and the 300 user tokens bill at $3.00 per 1M ($0.0009) for a total of $0.0015 per call. About a quarter of GPT-4o's input cost. The cache flips the comparison.

The context window

Claude Sonnet 4 has a 1,000,000-token context window. GPT-4o tops out at 128,000. For workloads that need to fit a large document, a multi-file codebase, or a long retrieved context into a single prompt, Claude Sonnet 4 is the only flagship-tier option of the two. Anthropic charges a premium for input beyond 200K tokens, so the very-long-context math is more nuanced than the headline rate suggests, but the 1M-token ceiling is real and GPT-4o can't match it.

For typical chat and short RAG workloads under 100K tokens, the context window is not the constraint and the headline rate is the right basis for comparison. The 1M-token context window is the differentiator when the workload actually uses it.

Five workloads, side by side

Five standard workload shapes, computed at the published rates with no caching applied:

Chat (1,000 in + 500 out)

GPT-4o: 1,000 × $2.50/M + 500 × $10.00/M = $0.0025 + $0.005 = $0.0075. Claude Sonnet 4: 1,000 × $3.00/M + 500 × $15.00/M = $0.003 + $0.0075 = $0.0105. Gap: $0.003 per call, GPT-4o cheaper. At 1M calls/month, GPT-4o is $7,500 vs Sonnet at $10,500.

RAG (5,000 in + 1,000 out)

GPT-4o: 5,000 × $2.50/M + 1,000 × $10.00/M = $0.0125 + $0.010 = $0.0225. Claude Sonnet 4: 5,000 × $3.00/M + 1,000 × $15.00/M = $0.015 + $0.015 = $0.030. Gap: $0.0075 per call. At 200K calls/month, GPT-4o is $4,500 vs Sonnet at $6,000.

Coding (3,000 in + 2,000 out)

GPT-4o: 3,000 × $2.50/M + 2,000 × $10.00/M = $0.0075 + $0.020 = $0.0275. Claude Sonnet 4: 3,000 × $3.00/M + 2,000 × $15.00/M = $0.009 + $0.030 = $0.039. Gap: $0.0115 per call. At 100K calls/month, GPT-4o is $2,750 vs Sonnet at $3,900.

Agentic (8,000 in + 4,000 out)

GPT-4o: 8,000 × $2.50/M + 4,000 × $10.00/M = $0.020 + $0.040 = $0.060. Claude Sonnet 4: 8,000 × $3.00/M + 4,000 × $15.00/M = $0.024 + $0.060 = $0.084. Gap: $0.024 per call. At 50K calls/month, GPT-4o is $3,000 vs Sonnet at $4,200.

Long context (50,000 in + 5,000 out)

GPT-4o: 50,000 × $2.50/M + 5,000 × $10.00/M = $0.125 + $0.050 = $0.175. Claude Sonnet 4: 50,000 × $3.00/M + 5,000 × $15.00/M = $0.150 + $0.075 = $0.225. Gap: $0.050 per call. But GPT-4o's context window caps at 128K, so a workload that needs more than 128K of input has to use Sonnet (or split the work across multiple GPT-4o calls, which adds latency).

What the math says

Across the five workloads, GPT-4o is consistently 20-30% cheaper on the headline rate. The gap widens on workloads that are output-heavy (the agentic workload is 40% output and shows a $0.024 gap) and shrinks on workloads that are input-heavy (the long-context workload shows a $0.050 gap but on a larger base cost). The cache multiplier, when applied to Claude Sonnet 4 on a cacheable prefix, flips the comparison on any workload with a substantial system prompt.

A simple decision rule. If your application has a long cacheable system prompt (more than 1,500 tokens), Claude Sonnet 4 with cache hits is the cheaper option. If it doesn't, GPT-4o is the cheaper option on every workload shape. If you need a context window above 128K tokens, Claude Sonnet 4 is the only option of the two. When the answer is "both"

Many production systems route traffic across both models. The cheap, fast path is GPT-4o for short replies, classification, and most chat traffic. The slow, capable path is Claude Sonnet 4 for long-context work, multi-file refactors, and any task where the cacheable prefix makes Sonnet cheaper than GPT-4o. The router is a small classifier that decides per request. The cost reduction article walks through the routing pattern in more detail.

Quality differences that matter for cost

The headline rate gap is meaningless if the cheaper model can't do the work. Two areas where the quality difference shows up directly in the bill: instruction-following and tool use. GPT-4o is more consistent at following complex multi-step instructions in a single prompt. Claude Sonnet 4 is more consistent at using tools correctly on the first call. For a coding agent that calls tools ten times per task, the difference between 8 and 9 successful first-try tool calls is the difference between ten API calls and eleven, a 10% bill increase from one extra tool call per task.

The right comparison here is on a workload the application actually runs, not on a benchmark. If the application is a chat assistant that does not call tools, GPT-4o is almost always cheaper at acceptable quality. If the application is a multi-tool agent, Claude Sonnet 4's stronger tool-use consistency can save more than the headline rate gap costs. The agentic workflow article walks through the multi-step cost math in detail.

Latency and throughput

Latency is not a direct line item on the bill, but it shapes how many requests the system can serve and how many concurrent users it can support. GPT-4o's median time-to-first-token on OpenRouter is in the 200-400ms range for short prompts. Claude Sonnet 4 sits in a similar range but with more variance on For long-context calls, the time-to-first-token for a 100K-token prompt is meaningfully higher on Sonnet 4 than on GPT-4o, because GPT-4o caps at 128K and isn't doing that workload at all.

For real-time chat, GPT-4o's lower variance on short prompts makes it the easier choice. For batch workloads where latency does not matter, throughput per dollar is the right metric, and Sonnet 4's higher output quality on long-context tasks can outweigh the rate gap.

Frequently asked questions

Is GPT-4o cheaper than Claude Sonnet 4?

On the rate card, yes. GPT-4o is $2.50 per 1M input and $10.00 per 1M output. Claude Sonnet 4 is $3.00 per 1M input and $15.00 per 1M output. The gap is 20% on input and 50% on output.

Does Claude's prompt cache make it cheaper in practice?

Often yes. Anthropic publishes a cached_input_per_1m at $0.30 for Sonnet 4, which is 10% of the input rate. OpenAI does not publish a cache rate for GPT-4o. For any workload with a long cacheable system prompt, Claude Sonnet 4 with cache hits can come out cheaper than GPT-4o on the same workload.

Which has the longer context window?

Claude Sonnet 4 supports a 1,000,000-token context window. GPT-4o tops out at 128,000 tokens. For long-document tasks, Claude Sonnet 4 is the only flagship-tier option of the two.

Which is better for coding?

Both are competitive on coding tasks. Claude Sonnet 4 has a longer context window and stronger performance on multi-file refactors. GPT-4o has the broader ecosystem of tooling and slightly faster latency on short completions.
