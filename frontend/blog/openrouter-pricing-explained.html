OpenRouter Pricing Explained: Cache, System Prompts, Tool Calls

One API key, one bill, 334 models. OpenRouter is an aggregator that passes through vendor pricing with a small platform fee. The bill is rarely a simple input-times-rate-times-output calculation. Cache rates, free tiers, per-tool-call surcharges, and reasoning fields each add a multiplier. There are enough quirks — caching, system prompt reuse, free tiers, per-tool-call surcharges — that a deployed bill is rarely the headline rate times the token counts. This article walks through what OpenRouter actually charges for, where the savings live, and where the surprises are.

What OpenRouter does

OpenRouter is an aggregator. The workflow is: sign up for one account, get one API key, and call any of the 334+ models behind it using the same OpenAI-compatible chat completions endpoint. Behind the scenes, OpenRouter routes the request to the upstream provider (Anthropic if the model is Claude, OpenAI if the model is GPT-4o, DeepSeek if the model is R1) and passes the response back. The pricing on each model is the upstream vendor's list price plus a small platform fee.

The platform fee is the part that surprises people who read OpenRouter's published rates next to the vendor's direct rates and see that they are close but not identical. For most models the difference is well under 10% of the bill. The trade-off is one bill, one key, and one consistent interface across every model family. Calling Anthropic, OpenAI, Google, DeepSeek, Mistral, Cohere, and Qwen directly means seven separate accounts, seven separate bills, and seven SDKs to maintain. OpenRouter consolidates that. Therefore OpenRouter is the right default for teams running multiple model families.

How the rate card is structured

The OpenRouter public endpoint at /api/v1/models returns a JSON object with the current rate for every model served. AI Cost Calculator refreshes its cache every six hours, so the numbers on each model page are never more than a few hours behind vendor list prices. The fields that matter for an estimate are input_per_1m, output_per_1m, cached_input_per_1m, reasoning_per_1m, and tool_call_cost. Multiply by the expected token counts, add a small buffer for cache misses, and the bill is the bill.

The free tier

In mid-2026, 26 models on the platform publish a $0 input and $0 output rate. The list includes Llama 3.2 3B Instruct, several Gemma 4 variants, Liquid's LFM 2.5 1.2B, and a handful of community-finetuned variants. They appear with a `:free` suffix on the slug and a $0.00 price on the model page.

Free models are not charity. The labs publish them for a reason: usually to drive adoption for a new model family, or because the inference cost is subsidized by an investor, or because the model is genuinely cheap to run (a 1.2B parameter model on a single GPU costs cents per million tokens). Treat free models as best-effort: rate limits are tighter than paid models, latency is variable, and the labs reserve the right to take them offline with little notice. They are great for prototyping, terrible for production. Clearly, the free tier is a prototyping tool, not a billing tool.

Cache pricing — the largest line item most teams miss

The largest single source of variance in a deployed LLM bill is the cache rate. Anthropic's Claude Sonnet 4 lists input at $3.00 per 1M tokens and cached input at $0.30 per 1M, a 10x discount. Claude Haiku 4.5 is $1.00 input, $0.10 cache. OpenAI's GPT-4.1 is $2.00 input and $0.50 cache. GPT-4o mini is $0.15 input and $0.075 cache, a 50% discount. Google's Gemini 2.5 Pro is $1.25 input and $0.125 cache.

The catch is that caching is prefix-based. The model has to see the same opening tokens to count them as cached. A 2,000-token system prompt sent on every call is a 2,000-token cache hit every time, as long as the prompt does not change. A 2,000-token system prompt with the user message appended to the end is a cache hit on the system portion only. If the user message comes first and the system prompt is appended to the end, the cache misses every time. AI Cost Calculator's calculator assumes cacheable structure by default. See the GPT-4o vs Claude Sonnet 4 comparison for an example of how cache hits change the per-call cost.

System prompts and the per-call tax

Most deployed prompts include a system prompt — persona, instructions, examples, tool definitions, retrieved context. A typical system prompt runs 500 to 3,000 tokens. That is a fixed per-call cost on every request the system makes.

Concrete math: a 2,000-token system prompt on GPT-4o at $2.50 per 1M is $0.005 per call. At 1 million calls per month, that system prompt alone is $5,000 per month. On Claude Sonnet 4 at $3.00 per 1M, the same system prompt costs $6,000 per month. If the prompt is cacheable and Anthropic's prefix cache hits, that drops to $600 per month. The structure of the prompt and where the cacheable block sits is the difference between a $5,000 line item and a $600 line item. See the system prompt tokens article for the full breakdown.

Tool calls

On most models in the catalog, the tool_call_cost field is $0. The tool call itself is free; what the bill charges for is the input tokens going into the call (the prompt plus the tool definitions) and the output tokens coming back (the model's response plus the tool call request). A handful of models publish non-zero tool call surcharges, usually older or specialty models. Those show up as a separate line item on the bill.

The cost driver for tool-using agents is not the tool call itself but the conversation growth. Each tool call adds the tool definition, the tool call request, and the tool result to the conversation history. By turn five of an agentic loop, the conversation is often 10x the size of the original prompt, and every subsequent call re-bills the entire history at the input rate. See the agentic workflow article for the multi-step math.

Reasoning tokens

Reasoning-capable models (101 of the 334 models in mid-2026) emit internal chain-of-thought tokens that are billed as output. OpenRouter surfaces the upstream vendor's reasoning field where the vendor publishes one. OpenAI's o-series bills reasoning at the output rate. Anthropic's extended thinking bills at the output rate. Google's Gemini 2.5 Pro publishes a reasoning_per_1m field that mirrors the output rate. DeepSeek's R1 publishes $0.70 input and $2.50 output, with reasoning in the output bucket.

The full breakdown of how reasoning tokens multiply the bill is in the reasoning token article. The short version: reasoning is a multiplier on the per-call cost that is invisible in the user-visible answer.

When NOT to use OpenRouter

Three scenarios where direct vendor access wins. First, when the upstream vendor has features that OpenRouter does not surface yet (Anthropic's prompt caching with explicit breakpoints behaves differently through OpenRouter than through the direct Anthropic API). Second, when the workload is latency-sensitive and the OpenRouter routing layer adds too much variance. Third, when negotiating an enterprise discount directly with the vendor — OpenRouter's pricing is the published rate card, not a negotiated rate.

The default for multi-model workloads is OpenRouter. The default for single-vendor workloads where the vendor has the relevant features is direct API. The default for latency-critical single-model workloads is direct API. The math and the routing change case by case.

Frequently asked questions

Does OpenRouter charge a markup over vendor pricing?

Yes. OpenRouter adds a small platform fee on top of the upstream vendor's list price, usually a few percent. For most models the difference between OpenRouter's rate and the vendor's direct rate is well under 10% of the bill.

Why are some models free on OpenRouter?

Labs sometimes publish preview or experimental models with a $0 rate to drive adoption, and a few community-trained open-weights models are routed through providers that subsidize the inference. In mid-2026 there are 26 free models on the platform. Treat them as best-effort: rate limits are tighter, latency is variable, and the labs can take them offline without notice.

How does OpenRouter's prompt caching work?

OpenRouter surfaces the upstream vendor's prompt cache. Anthropic publishes a cached_input_per_1m at 10% of the input rate. OpenAI publishes cache rates at 50% off input on supported models. Google's Gemini 2.5 family has implicit caching on prompts over a certain length. The cache applies only to the prefix, so the cacheable block must sit at the start of the prompt.

Do tool calls cost extra?

Most models surface a tool_call_cost field of $0. The tool result tokens are billed at the regular input rate when they come back. A handful of models publish a non-zero tool_call_cost; check the model's price table on AI Cost Calculator or the OpenRouter feed for the specific number.
