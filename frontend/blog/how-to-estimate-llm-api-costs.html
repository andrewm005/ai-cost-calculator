How to Estimate LLM API Costs: A Practical Guide

Estimating what a large language model will cost you before you ship is one of the most useful thirty-minute exercises in building with these APIs. A small number of multipliers, applied honestly, will tell you whether the bill at month-end will be $50 or $5,000. The trick is using real numbers from the model you're actually going to run, not the headline rate from a vendor blog post.

Step 1: Pick the model first

Every estimate starts with a model, because the input and output rates differ by orders of magnitude across the catalog. As of mid-2026, the spread between the cheapest and most expensive mainstream models on OpenRouter is roughly 1000x. Llama 3.1 8B Instruct runs $0.02 per 1M input tokens and $0.03 per 1M output tokens. Cohere's Command A sits at $2.50 input and $10.00 output. Anthropic's Claude Opus 4 is at $15.00 input and $75.00 output. OpenAI's o1 is $15.00 and $60.00. The model you pick is the dominant variable, and no amount of clever prompt engineering changes that.

A common trap is starting with the cheapest model and walking up the quality ladder until the answers are good enough. That's fine for a prototype, but for a production system, the right starting point is the model you would be happy to ship, then a cheaper fallback for the workloads where the flagship is overkill. The GPT-4o vs GPT-4o mini comparison is a useful template: a flagship for important calls, a mini for high-volume easy ones.

Step 2: Count tokens on a real call, not a minimum

Vendor pricing pages quote rates per million tokens, but no real call is one million tokens. Most chat messages are 200 to 2,000 input tokens and 100 to 1,000 output tokens. Retrieval-augmented generation runs in the thousands for input (the retrieved chunks) and hundreds for output (the final answer). Coding agents are heavier on both ends — the prompt carries tool definitions, retrieved code, and prior conversation history; the response carries generated code, tool call requests, and natural language explanations.

The right number to plug in is the token count of an actual call from your system, not the textbook minimum. Run a few representative conversations through your application, capture the input and output token counts from the API response, and take the median. Multiply that median by your expected monthly volume. If you don't have representative calls yet, the calculator's standard workload shapes are a reasonable starting point: 1,000 in + 500 out for a chat, 5,000 in + 1,000 out for RAG, 8,000 in + 4,000 out for an agent, 50,000 in + 5,000 out for a long context job.

Step 3: Apply the right multipliers

The headline rate is the easy part. The tricky part is the things the vendor charges for on top. Three multipliers show up on every production bill: cached input, reasoning tokens, and tool calls.

Cached input tokens are prompt prefixes the vendor recognizes as repeats. Anthropic, Google, and OpenAI all publish a cached_input_per_1m field, usually at 10% of the regular input rate. Claude Sonnet 4 lists input at $3.00 per 1M and cache at $0.30 per 1M, a 10x discount. If your prompt reuses a long system message, a tool definition block, or a retrieved document, the cache rate is what matters for most of your input tokens. A workflow that runs the same 4,000-token prompt prefix on every call effectively pays for those 4,000 tokens once and gets them cached for the next five minutes.

Reasoning tokens are the chain-of-thought tokens that reasoning models like OpenAI's o1 and o3, Anthropic's Claude (with extended thinking enabled), Google's Gemini 2.5 Pro, and DeepSeek's R1 emit internally before the user-visible answer. They are typically billed as output, sometimes at a higher rate. DeepSeek R1 publishes $0.70 input and $2.50 output per 1M, and the reasoning tokens fall into that output bucket. If your agentic workflow is making three or four reasoning calls per user request, reasoning tokens can dominate the bill, sometimes without the user seeing any extra content.

Tool calls are usually a flat fee per call on OpenRouter (currently $0 for most models, but not all). OpenAI charges for individual tools through its Assistants API. Anthropic charges nothing per call but the tool result tokens go through the regular input rate. Either way, multi-step agentic workflows multiply the cost: ten tool calls in a single user request is ten input round-trips through the model's context window.

Step 4: Sanity-check the math

The four-line sanity check is: pick the model, pick a workload shape, multiply by the per-million rates, compare to the price the calculator spits out. If the numbers are within 10%, your estimate is probably right. If they're off by 5x, something is wrong — usually an undercount of output tokens, or a missed reasoning tier, or a system prompt that ships 1,500 tokens of instructions on every call without being cached.

Two concrete examples. First, a chatbot on GPT-4o with a 500-token system prompt, a 300-token user message, and a 250-token response. Input is 800 tokens at $2.50 per 1M ($0.002), output is 250 tokens at $10.00 per 1M ($0.0025), total $0.0045 per message. At 100,000 messages per month, that's $450. Second, an agentic coding assistant running on Claude Sonnet 4 with an 8,000-token prompt (system prompt + retrieved files + tool definitions), a 4,000-token response including several tool calls, and 10 tool invocations. Input is $0.024, output is $0.060, total $0.084 per run. At 5,000 runs per month, that's $420. Both are reasonable working numbers.

Common mistakes

The most common mistake is forgetting the system prompt. A long system prompt with examples, persona instructions, and tool definitions can easily hit 1,000 to 3,000 tokens, and it ships on every call. If you're using a 3,000-token system prompt on a million chat messages a month, that's 3 billion input tokens, or $7,500 at GPT-4o rates and $750 at Claude Sonnet 4 rates. The system prompt is also the easiest place to apply cache: most vendors will cache the prefix automatically if you structure the request to put the cacheable part first.

The second most common mistake is forgetting that streaming changes nothing about billing. Some teams assume streaming is cheaper because the user sees tokens as they arrive. It's not — you pay for the same number of output tokens regardless of whether they arrive in a single batch or one at a time.

The third is forgetting that long context has its own surcharge on some providers. OpenAI charges double the rate for input tokens beyond 200K on GPT-4.1. Anthropic's Sonnet 4 has a 1M-token context window but charges a premium for prompts longer than 200K tokens. Most chat and agent workloads stay well under those thresholds, but if you're summarizing a long document or processing a long code file, the second half of the context can cost more than the first half.

What to do with the number

Once you have a real estimate, the right next step is to multiply by your expected volume and compare to the revenue or value the application generates. If the per-call cost is $0.005 and the average user generates 100 calls per month, that's $0.50 per user per month for inference. A subscription at $5/month covers it with margin. A subscription at $0.99 does not. If the application is ad-supported, you need to know the effective CPM per inference call. If it's a one-shot tool, you need the number before you can price the SKU.

Frequently asked questions

How do I estimate the cost of an LLM API call before shipping?

Pick the model, count the input and output tokens for one realistic call (not the minimum), multiply by the per-million rates for that model, then scale by expected monthly volume. A calculator like AI Cost Calculator handles the multipliers (caching, reasoning tokens, tool calls) that vendors charge separately.

What is the difference between input and output tokens for billing?

Input tokens are the prompt you send (system prompt, user message, retrieved context). Output tokens are the model's response. Output is typically 3 to 5 times more expensive than input because generation is more compute-intensive. On GPT-4o, input is $2.50 per 1M tokens and output is $10.00 per 1M tokens.

Do reasoning tokens count as input or output?

Reasoning tokens are billed as output on most providers, but at a different rate tier. OpenAI's o1 and o3, Anthropic's extended-thinking Claude, Google's Gemini 2.5 Pro, and DeepSeek-R1 all publish a separate reasoning_per_1m field. The chain-of-thought tokens are typically billed at the output rate or higher.

How much does a single chat message cost on a typical LLM?

A short chat message (around 1,000 input tokens and 500 output tokens) costs roughly $0.0075 on GPT-4o, $0.003 on Claude Sonnet 4, $0.0018 on Gemini 2.5 Flash, and $0.00026 on Llama 3.1 8B. The same exchange on Claude Opus 4 costs about $0.0450, and on OpenAI o1 around $0.0375.
