About GLM 4.6V

GLM 4.6V is a mid-tier model in the Z.AI lineup, priced at $0.30 per 1M input tokens and $0.90 per 1M output tokens. It is the kind of model most teams end up defaulting to once they have measured their workloads: cheap enough to run at moderate volume, capable enough that the answer quality does not require a flagship. Z.AI generally publishes a budget option below this model and a flagship above it, and GLM 4.6V is the middle step in that ladder.

At $0.30 per 1M input tokens and $0.90 per 1M output tokens, the cost of a single call is dominated by output unless the workload is very output-heavy or very input-heavy in unusual ways. Output is roughly 3.0x the input cost, which is the typical shape for chat-tuned models where generation is the expensive direction. The context window is about 131K tokens, which covers most single-turn chat and short retrieval tasks but requires chunking for long documents. Z.AI's pricing for this model is published through OpenRouter and refreshed every six hours; this page reads the live values, so the numbers here match what you would see if you called the OpenRouter models endpoint directly.

What makes GLM 4.6V different

Z.AI (Zhipu AI, founded 2019 in Beijing) ships the GLM line as their primary frontier family. GLM stands for General Language Model, and the family has been the company's flagship open-weights push since the GLM-130B release in 2022. The current GLM 4.6V sits inside that lineage: same OpenRouter endpoint as any other model, but tuned and post-trained against Zhipu's own data, with multimodal variants (the GLM 4.6V family often includes a -v version) handling images and the text-only versions hitting lower per-token prices. For US-headquartered teams, GLM is the less-marketed sibling of Qwen and DeepSeek in the Chinese open-weights space; for China-headquartered teams and for multilingual workloads, it is one of the default picks.

GLM 4.6V is built for a mid-tier GLM 5 family price point, balanced for chat, RAG, and agent loops where cost matters but answer quality is non-negotiable. It fits well in Chinese language workloads where Zhipu's training data gives it an edge over US-trained models on idiomatic Chinese, in document-understanding pipelines that need multimodal input without paying GPT-4o prices, and in any workload that wants open-weights lineage (Zhipu has historically published GLM weights under a permissive license) without giving up production-grade API availability on OpenRouter.

Frequently asked questions

How much does GLM 4.6V cost per 1M tokens?

GLM 4.6V costs $0.30 per 1M input tokens and $0.90 per 1M output tokens on OpenRouter. A typical chat run (1,000 input, 500 output) costs about $0.0008; a long-context job (50,000 input, 5,000 output) costs about $0.019.

What is the context window for GLM 4.6V?

The context window for GLM 4.6V is 131,072 tokens, which covers most single-turn chat and short retrieval tasks. Anything longer has to be chunked or summarized before it fits.

Does GLM 4.6V support tool use / function calling?

OpenRouter does not publish a tool-use flag for GLM 4.6V on this listing, so assume no and validate in your own integration. If the upstream feed adds the flag, the next six-hourly refresh will surface it here.

Is GLM 4.6V cheaper than GLM 4.7 Flash?

No — GLM 4.6V is more expensive than GLM 4.7 Flash on input by about 80% ($0.30 vs $0.06 per 1M). Output is $0.90 per 1M versus $0.40 for GLM 4.7 Flash. See the GLM 4.7 Flash model page for the full price table and workload examples.

How much does it cost to run GLM 4.6V for chatbot?

A typical chatbot run (1,000 input + 500 output) on GLM 4.6V costs about $0.000750 per call. At 10,000 runs/month that is roughly $7.50. The bulk of that is output at $0.9000 per 1M tokens.
