About Sonar

Sonar is a mid-tier model in the Perplexity lineup, priced at $1.00 per 1M input tokens and $1.00 per 1M output tokens. It is the kind of model most teams end up defaulting to once they have measured their workloads: cheap enough to run at moderate volume, capable enough that the answer quality does not require a flagship. Perplexity generally publishes a budget option below this model and a flagship above it, and Sonar is the middle step in that ladder.

At $1.00 per 1M input tokens and $1.00 per 1M output tokens, the cost of a single call is dominated by output unless the workload is very output-heavy or very input-heavy in unusual ways. Output is priced close to input (1.0x), which is the shape of models that lean toward balanced input/output workloads — analysis, summarization, and similar tasks where the response is comparable in size to the prompt. The context window is about 127K tokens, which covers most single-turn chat and short retrieval tasks but requires chunking for long documents. Perplexity's pricing for this model is published through OpenRouter and refreshed every six hours; this page reads the live values, so the numbers here match what you would see if you called the OpenRouter models endpoint directly.

Sonar is the right pick when the workload needs more than a budget model but does not justify a flagship. It fits well in: the everyday chat and document work that fills most production traffic — emails, summaries, structured extraction, mid-volume agents where the answer quality is good enough and the per-call cost is reasonable, tool-calling workflows where the model's instruction following is the main requirement, and long-running assistants where the cost adds up over time and a budget tier would be too thin.

It is not the right pick for: the hardest analytical questions where a flagship is the only model that can solve them — pay the extra for the best answer, not the second-best, or the cheapest possible work where the model is overkill — a budget model from the same provider will produce a similar enough answer at a fraction of the cost. For those workloads, the right move is to pick a model from a different tier in the same provider lineup, or to use a different provider entirely.

When to use Sonar

  • Everyday chat and document work that fills most production traffic
  • Mid-volume agents where answer quality matters but the bill is still reasonable
  • Tool-calling workflows where instruction following is the main requirement
  • Long-running assistants where cost adds up over time

When NOT to use Sonar

  • Cheapest possible production traffic where the per-call cost dominates — step down to a budget-tier model from the same provider and save 70-90% on input tokens at a measurable but usually acceptable quality drop.
  • Hardest analytical questions where a flagship model would produce a measurably stronger answer — Sonar is calibrated for the middle of the quality curve, not the long tail.

Frequently asked questions

How much does Sonar cost per 1M tokens?

Sonar costs $1.00 per 1M input tokens and $1.00 per 1M output tokens on OpenRouter. A typical chat run (1,000 input, 500 output) costs about $0.0015; a long-context job (50,000 input, 5,000 output) costs about $0.055.

What is the context window for Sonar?

The context window for Sonar is 127,072 tokens, which covers most single-turn chat and short retrieval tasks. Anything longer has to be chunked or summarized before it fits.

Does Sonar support tool use / function calling?

OpenRouter does not publish a tool-use flag for Sonar on this listing, so assume no and validate in your own integration. If the upstream feed adds the flag, the next six-hourly refresh will surface it here.

Is Sonar cheaper than MiniMax M2?

No — Sonar is more expensive than MiniMax M2 on input by about 74% ($1.00 vs $0.26 per 1M). Output is $1.00 per 1M versus $1.00 for MiniMax M2. See the MiniMax M2 model page for the full price table and workload examples.

How much does it cost to run Sonar for chatbot?

A typical chatbot run (1,000 input + 500 output) on Sonar costs about $0.0015 per call. At 10,000 runs/month that is roughly $15.00. The bulk of that is input at $1.00 per 1M tokens.
