About GPT-4 Turbo

GPT-4 Turbo is priced in the premium band at $10.00 per 1M input tokens and $30.00 per 1M output tokens. It is aimed at workloads where the model is doing real work — long-form analysis, multi-turn agents, code reviews of nontrivial pull requests — and where cheaper models from the same provider would not produce a strong enough answer. The pricing reflects that role: not the most expensive offering, but not a budget pick either, and the right default for work that needs the provider's better-than-mid-tier capability.

At $10.00 per 1M input tokens and $30.00 per 1M output tokens, the cost of a single call is dominated by output unless the workload is very output-heavy or very input-heavy in unusual ways. Output is roughly 3.0x the input cost, which is the typical shape for chat-tuned models where generation is the expensive direction. The context window is about 128K tokens, which covers most single-turn chat and short retrieval tasks but requires chunking for long documents. OpenAI's pricing for this model is published through OpenRouter and refreshed every six hours; this page reads the live values, so the numbers here match what you would see if you called the OpenRouter models endpoint directly.

GPT-4 Turbo is the right pick when the answer quality is worth paying for. It fits well in: workloads where the answer quality justifies the cost: long-form synthesis, complex code reviews, and high-stakes analysis, workloads where a wrong answer is expensive — refactor decisions, security review, and similar jobs, production agents that need consistent quality across a wide range of inputs, where the flagship's lower variance is worth the price, and long-context tasks where the model's ability to use the full window is the actual differentiator.

It is not the right pick for: high-volume work where the per-call cost would dominate the bill — for that, drop to a budget model in the same provider lineup and accept slightly weaker answers, or throwaway or experimental runs where the answer quality is not the point — cheaper models will produce similar results for those tasks. For those workloads, the right move is to pick a model from a different tier in the same provider lineup, or to use a different provider entirely.

When to use GPT-4 Turbo

  • Long-form synthesis, complex code reviews, and high-stakes analysis
  • Workloads where a wrong answer is expensive
  • Production agents that need consistent quality across a wide range of inputs
  • Long-context tasks where the model's full window is the differentiator

When NOT to use GPT-4 Turbo

  • Workloads where gpt-oss-120b (free) ($0.00/1M input) would produce a comparable answer at lower cost — premium tier is the right call when the quality gap is visible in your eval, not when it is theoretical.
  • Bulk extraction, classification, or routing pipelines where the per-call cost dominates the bill — a cheap-tier model from the same provider will handle the same workload at 5-10x lower cost than GPT-4 Turbo.

Frequently asked questions

How much does GPT-4 Turbo cost per 1M tokens?

GPT-4 Turbo costs $10.00 per 1M input tokens and $30.00 per 1M output tokens on OpenRouter. A typical chat run (1,000 input, 500 output) costs about $0.0250; a long-context job (50,000 input, 5,000 output) costs about $0.650.

What is the context window for GPT-4 Turbo?

The context window for GPT-4 Turbo is 128,000 tokens, which covers most single-turn chat and short retrieval tasks. Anything longer has to be chunked or summarized before it fits.

Does GPT-4 Turbo support tool use / function calling?

OpenRouter does not publish a tool-use flag for GPT-4 Turbo on this listing, so assume no and validate in your own integration. If the upstream feed adds the flag, the next six-hourly refresh will surface it here.

Is GPT-4 Turbo cheaper than gpt-oss-120b (free)?

No — GPT-4 Turbo is more expensive than gpt-oss-120b (free) on input by about 100% ($10.00 vs $0.00 per 1M). Output is $30.00 per 1M versus $0.00 for gpt-oss-120b (free). See the gpt-oss-120b (free) model page for the full price table and workload examples.

How much does it cost to run GPT-4 Turbo for coding agent?

A typical coding agent run (8,000 input + 4,000 output + 5 tool calls) on GPT-4 Turbo costs about $0.2000 per call. At 10,000 runs/month that is roughly $2,000.00. The bulk of that is output at $30.00 per 1M tokens.
