About Claude Sonnet 4

Claude Sonnet 4 is priced in the premium band at $3.00 per 1M input tokens and $15.00 per 1M output tokens. It is aimed at workloads where the model is doing real work — long-form analysis, multi-turn agents, code reviews of nontrivial pull requests — and where cheaper models from the same provider would not produce a strong enough answer. The pricing reflects that role: not the most expensive offering, but not a budget pick either, and the right default for work that needs the provider's better-than-mid-tier capability.

At $3.00 per 1M input tokens and $15.00 per 1M output tokens, the cost of a single call is dominated by output unless the workload is very output-heavy or very input-heavy in unusual ways. Output is priced noticeably above input (5.0x), reflecting the typical chat-shape where the model spends more compute on generation than on reading the prompt. The context window is about 1M tokens, large enough to hold a substantial codebase, a long document, or an extended multi-turn conversation without truncation. Anthropic's pricing for this model is published through OpenRouter and refreshed every six hours; this page reads the live values, so the numbers here match what you would see if you called the OpenRouter models endpoint directly.

Claude Sonnet 4 is the right pick when the answer quality is worth paying for. It fits well in: workloads where the answer quality justifies the cost: long-form synthesis, complex code reviews, and high-stakes analysis, workloads where a wrong answer is expensive — refactor decisions, security review, and similar jobs, production agents that need consistent quality across a wide range of inputs, where the flagship's lower variance is worth the price, and long-context tasks where the model's ability to use the full window is the actual differentiator.

It is not the right pick for: high-volume work where the per-call cost would dominate the bill — for that, drop to a budget model in the same provider lineup and accept slightly weaker answers, or throwaway or experimental runs where the answer quality is not the point — cheaper models will produce similar results for those tasks. For those workloads, the right move is to pick a model from a different tier in the same provider lineup, or to use a different provider entirely.

When to use Claude Sonnet 4

  • Long-form synthesis, complex code reviews, and high-stakes analysis
  • Workloads where a wrong answer is expensive
  • Production agents that need consistent quality across a wide range of inputs
  • Long-context tasks where the model's full window is the differentiator

When NOT to use Claude Sonnet 4

  • Workloads where Claude 3 Haiku ($0.25/1M input) would produce a comparable answer at lower cost — premium tier is the right call when the quality gap is visible in your eval, not when it is theoretical.
  • Bulk extraction, classification, or routing pipelines where the per-call cost dominates the bill — a cheap-tier model from the same provider will handle the same workload at 5-10x lower cost than Claude Sonnet 4.
