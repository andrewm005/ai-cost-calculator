About GPT-3.5 Turbo

GPT-3.5 Turbo is a mid-tier model in the OpenAI lineup, priced at $0.50 per 1M input tokens and $1.50 per 1M output tokens. It is the kind of model most teams end up defaulting to once they have measured their workloads: cheap enough to run at moderate volume, capable enough that the answer quality does not require a flagship. OpenAI generally publishes a budget option below this model and a flagship above it, and GPT-3.5 Turbo is the middle step in that ladder.

At $0.50 per 1M input tokens and $1.50 per 1M output tokens, the cost of a single call is dominated by output unless the workload is very output-heavy or very input-heavy in unusual ways. Output is roughly 3.0x the input cost, which is the typical shape for chat-tuned models where generation is the expensive direction. The context window is about 16K tokens, which covers most single-turn chat and short retrieval tasks but requires chunking for long documents. OpenAI's pricing for this model is published through OpenRouter and refreshed every six hours; this page reads the live values, so the numbers here match what you would see if you called the OpenRouter models endpoint directly.

GPT-3.5 Turbo is the right pick when the workload needs more than a budget model but does not justify a flagship. It fits well in: the everyday chat and document work that fills most production traffic — emails, summaries, structured extraction, mid-volume agents where the answer quality is good enough and the per-call cost is reasonable, tool-calling workflows where the model's instruction following is the main requirement, and long-running assistants where the cost adds up over time and a budget tier would be too thin.

It is not the right pick for: the hardest analytical questions where a flagship is the only model that can solve them — pay the extra for the best answer, not the second-best, or the cheapest possible work where the model is overkill — a budget model from the same provider will produce a similar enough answer at a fraction of the cost. For those workloads, the right move is to pick a model from a different tier in the same provider lineup, or to use a different provider entirely.

When to use GPT-3.5 Turbo

  • Everyday chat and document work that fills most production traffic
  • Mid-volume agents where answer quality matters but the bill is still reasonable
  • Tool-calling workflows where instruction following is the main requirement
  • Long-running assistants where cost adds up over time

When NOT to use GPT-3.5 Turbo

  • Cheapest possible production traffic where the per-call cost dominates — step down to a budget-tier model from the same provider and save 70-90% on input tokens at a measurable but usually acceptable quality drop.
  • Hardest analytical questions where a flagship model would produce a measurably stronger answer — GPT-3.5 Turbo is calibrated for the middle of the quality curve, not the long tail.

The context window for GPT-3.5 Turbo is 16,385 tokens, which covers most single-turn chat and short retrieval tasks. Anything longer has to be chunked or summarized before it fits.
