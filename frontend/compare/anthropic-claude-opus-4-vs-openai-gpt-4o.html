Hidden costs the table doesn't show

Claude Opus 4 — prompt caching is the biggest single discount in this comparison. Opus 4 cache reads cost $1.50 per 1M tokens (10x cheaper than the $15.00 input rate) and cache writes cost $18.75 per 1M (a 25% premium on the first hit). A production workload with a 5,000-token system prompt and 70–90% cache hits drops Opus's effective input cost to roughly $2.85–$4.50 per 1M — still above GPT-4o's $2.50 headline, but the gap closes from 6x to roughly 1.5–2x.

GPT-4o — image inputs cost extra (the table only prices text); tool-call overhead applies on tool-capable endpoints but is a flat per-call figure, not per-token. OpenAI's hosted API also offers batch API discounts (~50% off) that this comparison doesn't apply. If you batch GPT-4o, the gap to Opus widens from 7x to roughly 14x.

Verdict

Pick Claude Opus 4 for frontier reasoning tasks — long-horizon agentic coding, multi-file refactors, and decisions where a wrong answer is expensive. Opus 4 is the strongest model in this tier on the hardest problems and the per-call cost is justified when the call is rare and the answer is high-stakes.

Pick GPT-4o for the everyday coding workflow — the 7x price gap is real and GPT-4o is the right default for most production traffic. At 1M coding calls/month, Opus 4 costs $195,000 and GPT-4o costs $27,500 — the difference is two engineers.

They're a wash for short single-turn chat with no cache, no tool loop, and no system prompt — rare in production but worth noting if you have a prototype with no cacheable prompt.

How to read this comparison

The numbers above are the per-call cost of running the same workload — same input token count, same output token count — against each model. We don't add per-request fees, batch discounts, or commitment pricing; the table is the headline on-demand rate from OpenRouter's feed. If your workload uses cached inputs or reasoning tokens, both are priced separately in the main calculator (open the Advanced panel).
