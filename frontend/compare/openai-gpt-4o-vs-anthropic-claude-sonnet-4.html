Hidden costs the table doesn't show

GPT-4o — image inputs cost extra (the table only prices text); tool-call overhead applies on tool-capable endpoints but is a flat per-call figure, not a per-token one. If your workload is vision-heavy, model the image line separately.

Claude Sonnet 4 — prompt caching is the biggest single discount in this comparison. Sonnet 4 cache reads cost $0.30 per 1M tokens (10x cheaper than the $3.00 input rate) and cache writes cost $3.75 per 1M (a 25% premium on the first hit). A production system with a 2,000-token system prompt that sees 100k requests/month typically hits the cache on 70–90% of input tokens, which flips Sonnet's effective input cost to roughly $0.55–$0.85 per 1M — below GPT-4o's $2.50 headline.

Verdict

Pick GPT-4o if your workload is short prompts and short answers with no cacheable system prompt — the 40% price gap on every request is real, and GPT-4o's tool/vision story is the better default for new builds.

Pick Claude Sonnet 4 if you ship a system prompt above ~1,500 tokens that repeats across requests — cache flips the math, and Sonnet 4's tool-use instruction-following is the strongest in this tier for agentic workflows.

They're a wash for one-off single-turn chat where there is no cache, no tool loop, and no system prompt — both models are sub-cent per call and the price difference is small relative to the engineering cost.

How to read this comparison

The numbers above are the per-call cost of running the same workload — same input token count, same output token count — against each model. We don't add per-request fees, batch discounts, or commitment pricing; the table is the headline on-demand rate from OpenRouter's feed. If your workload uses cached inputs or reasoning tokens, both are priced separately in the main calculator (open the Advanced panel).
