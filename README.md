# AI Cost Calculator

Free, no-signup calculator that estimates the cost of running AI tasks
across 80+ models. Live pricing from OpenRouter's public catalog,
refreshed every six hours. No tracking, no cookies, no analytics.

**Live site:** https://aicostcalculator.net

The calculator runs entirely in your browser. The backend serves the
price list; the math runs client-side. You can verify the source code
in this repo to confirm the math matches what the page shows.

## What the site does

Type in a model, a workload size, and a few workload knobs. Get a
dollar figure for the call, plus a breakdown by input, output, cached
input, reasoning, tools, and images.

80 models across 11 providers (OpenAI, Anthropic, Google, Meta, Mistral,
DeepSeek, xAI, Cohere, Qwen, Z.AI, plus specialty models). Each model
has its own page with pricing, workload examples, and FAQ.

You can compare two models side-by-side at `/compare/` (13 combinations),
read deeper explanations at `/blog/` (11 articles), and check whether
the backend is currently healthy at `/status.html`.

## Stack

- **Frontend:** static HTML, CSS, and vanilla JavaScript in `frontend/`.
  No build step. Served by Cloudflare Pages at the edge.
- **Backend:** Python (FastAPI) in `app/` for local dev / CI parity tests.
  Mirror TypeScript backend (Hono on Node) in `worker/` for production.
- **Pricing data:** OpenRouter's public API at
  `https://openrouter.ai/api/v1/models`, cached in `config/openrouter.json`.
  Hand-curated entries for direct-API models in `config/pricing.json`.

## For end users

If you just want to use the calculator, go to
https://aicostcalculator.net. Bookmark the `/status.html` page to verify
the site is up.

If you want to understand how a number was calculated, click the
"show assumptions" disclosure on any result panel. It shows every
multiplier that was applied.

If you want to report a wrong price, file an issue with the model name
and the price you saw. We re-fetch from OpenRouter every six hours,
but a stale price is a real bug worth fixing.

## For developers

This repo contains three runnable components:

### Frontend (`frontend/`)

Static files. Open `index.html` in any browser, or serve the directory
with any static server:

```bash
cd frontend
python3 -m http.server 3018
# http://localhost:3018
```

The frontend talks to a backend at the URL configured in `app.js:5`
(default `http://10.10.10.205:8002` for local testing, empty string for
production same-origin).

### Python backend (`app/`)

FastAPI on Python 3.14. Reads pricing from `config/pricing.json` and
`config/openrouter.json`, refreshes from OpenRouter every six hours.

```bash
.venv/bin/python -m uvicorn app.main:app --host 0.0.0.0 --port 8001
# http://localhost:8001/docs for Swagger UI
```

Tests:

```bash
.venv/bin/python -m pytest tests/
# 102 tests
```

### TypeScript backend (`worker/`)

Hono on Node 20. Same endpoints as the Python backend, byte-for-byte
equivalent calculator math. Runs the same parity test suite plus a
vitest mirror of the pytest suite.

```bash
cd worker
npm install
npm test                                    # 58 vitest tests
PORT=8002 node --import tsx src/server.ts    # boots on :8002
```

Calculator parity between Python and TypeScript:

```bash
python3 worker/tests/parity_test.py
# 10/10 payloads OK, delta < 1e-6
```

## Pricing data

Hand-curated entries live in `config/pricing.json`. The OpenRouter feed
goes to `config/openrouter.json` and is auto-generated by the backend
every six hours.

To force a refresh:

```bash
curl -X POST http://localhost:8001/admin/openrouter/refresh
```

To inspect one model's pricing:

```bash
curl http://localhost:8001/models/openai/gpt-4o
```

## Repo conventions

- The `main` branch is what gets deployed. Anything on `main` is live.
- Pricing numbers in `config/` are real data. Don't hand-edit the
  OpenRouter cache file.
- The 80 model pages and 11 blog posts are static HTML. Generated by
  scripts in `scripts/`. If you change a generator, re-run it before
  committing.
